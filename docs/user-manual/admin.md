# 管理员手册

本手册帮助管理员（ADMIN 角色）使用 Epoch Judge 系统进行系统配置和管理。

## 1. 管理后台概览

管理员登录后，导航栏右侧会出现「管理后台」入口。管理后台包含以下标签页：

| 标签 | 功能 |
|------|------|
| 配置 | 系统全局配置项 |
| 用户 | 用户账号管理 |
| 题目 | 题目管理 |
| 比赛 | 比赛管理 |
| 公告 | 公告管理 |
| 重判 | 提交重判 |

## 2. 系统配置

### 查看配置

1. 进入「管理后台」→「配置」标签页
2. 查看所有配置项及其当前值

### 修改配置

1. 找到要修改的配置项
2. 点击编辑按钮
3. 输入新的配置值
4. 点击保存

> ⚠️ 修改配置后可能需要重启服务才能生效。

## 3. 用户管理

### 查看用户列表

1. 进入「管理后台」→「用户」标签页
2. 查看所有用户信息：
   - 用户名
   - 邮箱
   - 昵称
   - 角色
   - 创建时间

### 创建用户

1. 点击「新建用户」按钮
2. 填写用户信息：
   - **用户名**（必填）：登录标识，创建后不可修改
   - **密码**（必填）：至少 6 位
   - **邮箱**（选填）
   - **昵称**（选填）：最多 64 个字符
   - **角色**（必填）：
     - `USER`：普通学生
     - `PROBLEM_EDITOR`：教师，可管理题目
     - `ADMIN`：管理员，拥有所有权限
3. 点击「确定」创建

### 编辑用户

1. 在用户列表中找到目标用户
2. 点击「编辑」按钮
3. 修改用户信息（用户名不可修改）
4. 如需修改密码，输入新密码；留空则不修改
5. 点击「确定」保存

### 重置密码

1. 在用户列表中找到目标用户
2. 点击「重置密码」按钮
3. 确认重置操作
4. 通知用户使用原用户名在注册页设置新密码

> 💡 重置密码后，用户需要访问注册页面，输入原用户名来设置新密码。

### 删除用户

1. 在用户列表中找到目标用户
2. 点击「删除」按钮
3. 确认删除操作

> ⚠️ 删除用户操作不可撤销，请谨慎操作。

### 批量导入用户

1. 点击「下载模板」获取 CSV 模板文件
2. 按模板格式填写用户信息
3. 点击「批量导入 CSV」上传文件
4. 系统自动创建用户账号

#### CSV 模板格式

```csv
username,password,email,displayName
student001,password123,student001@example.com,张三
student002,password456,student002@example.com,李四
teacher001,teacher123,teacher001@example.com,王老师
```

## 4. 题目管理

### 查看题目列表

1. 进入「管理后台」→「题目」标签页
2. 查看所有题目信息：
   - 编号
   - 标题
   - 可见性
   - 难度
   - 时间限制
   - 内存限制
   - 标签

### 创建题目

1. 点击「新建题目」按钮
2. 填写基本信息：
   - **标题**（必填）
   - **可见性**：PUBLIC（公开）或 PRIVATE（私有）
   - **标签**（选填）
3. 点击「创建」
4. 在编辑页面补充详细信息：
   - **题目描述**：支持 Markdown 格式
   - **难度**：0-5000 的数值
   - **时间限制**：毫秒
   - **内存限制**：KB
5. 切换到「测试用例」标签添加测试点
6. 保存

### 编辑题目

1. 在题目列表中点击「编辑」
2. 修改题目信息
3. 保存

### 导出题目

1. 在题目列表中点击「导出」
2. 下载 JSON 格式的题目文件（包含题目信息和测试用例）

## 5. 比赛管理

### 查看比赛列表

1. 进入「管理后台」→「比赛」标签页
2. 查看所有比赛信息：
   - ID
   - 标题
   - 评测模式
   - 可见性
   - 时间
   - 题目数量

### 创建比赛

1. 点击「新建比赛」按钮
2. 填写比赛信息：

| 字段 | 说明 |
|------|------|
| 标题 | 比赛名称 |
| 说明 | 比赛描述（支持 Markdown） |
| 可见性 | PUBLIC / PRIVATE |
| 评测模式 | ACM / OI |
| 开始时间 | 精确到分钟 |
| 结束时间 | 可设置时长或指定时刻 |
| 封榜 | 结束前多久封榜（可选） |
| 比赛密码 | 访问密码（可选） |
| 比赛题目 | 选择题目，拖拽排序 |

3. 点击「保存」

### 编辑比赛

1. 在比赛列表中点击「编辑」
2. 修改比赛信息
3. 保存

### 管理比赛报名

1. 在比赛列表中点击「报名/打星」
2. 查看已报名学生列表
3. 设置打星队伍（不参与排名）

### 删除比赛

1. 在比赛列表中点击「删除」
2. 确认删除

> ⚠️ 已有提交记录的比赛无法删除。

## 7. 重判管理

### 功能说明

重判功能用于在修改题目或测试用例后，重新评测已提交的代码。

### 操作步骤

1. 进入「管理后台」→「重判」标签页
2. 选择重判范围：
   - **按提交**：选择特定提交记录
   - **按题目**：选择特定题目的所有提交
   - **按比赛**：选择特定比赛的所有提交
3. 可选：按状态筛选（AC、WA、TLE 等）
4. 点击「预览」查看影响范围
5. 确认后点击「重判」执行

### 重判说明

- 重判会重新评测选定的所有提交
- 预览会显示符合条件的提交数量和跳过的提交
- 执行后显示已排队和跳过的提交数量

> ⚠️ 重判操作可能需要较长时间，请耐心等待。

## 8. 判题节点管理

### 查看节点状态

1. 进入「管理后台」→「配置」标签页
2. 查看判题节点列表：
   - 节点名称
   - 在线状态
   - 最后心跳时间
   - 并发数

### 节点说明

- 判题节点负责执行代码评测
- 节点通过心跳机制保持在线状态
- 可以部署多个节点以提高评测吞吐量

> 💡 判题节点的部署请参考 [部署指南](../deploy.md)。

## 常见问题

### Q: 如何给教师开通权限？
A: 在用户管理中编辑该用户，将角色改为 `PROBLEM_EDITOR`。

### Q: 如何批量创建学生账号？
A: 使用「批量导入 CSV」功能，下载模板后填写学生信息并上传。

### Q: 比赛结束后还能修改题目吗？
A: 可以修改，但已有的提交记录不会自动更新，需要手动重判。

### Q: 如何备份系统数据？
A: 可以通过导出题目功能备份题目数据，用户数据需要通过数据库备份。

### Q: 判题节点离线怎么办？
A: 检查判题节点服务器是否正常运行，查看日志排查问题。可参考部署文档重新部署节点。
