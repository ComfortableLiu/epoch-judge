## Why

题库与比赛题目目前缺少「未公开题在比赛期间不外泄」的访问控制：新建题目默认可被公开浏览，比赛绑定的隐藏题也可能通过题库列表暴露。同时管理端新建比赛的交互（slug、时间、题目多选）不符合运维习惯，且无法设置进入比赛的密码。

## What Changes

- **题目访问防护**：新建题目默认 `PRIVATE`；记录创建者；全局仅创建者、管理员（及授权编辑角色）可访问；被某比赛关联的题目在该比赛页面内对符合比赛准入的用户可见；若题目仍关联**未结束**的比赛，则无论 `visibility` 为何，在题库等全局入口仅创建者与管理员可见（比赛内上下文除外）。
- **比赛标识**：移除比赛 `slug` 与用户填写的 Slug 字段；比赛主键改为自增数字 ID，前台/管理端路由与 API 以数字 ID 为准（**BREAKING**：`/contests/:slug` → `/contests/:id`）。
- **管理端比赛表单**：开始时间精确到分钟；结束时间支持「相对时长（数值+单位）」与「绝对时间」双向联动，默认结束 = 开始 + 3 小时；封榜支持「距结束 x 分钟/小时」相对设置。
- **比赛题目编辑**：管理端以可拖拽排序的列表维护题目（增删、调整顺序），顺序仍映射 A/B/C 标号。
- **比赛密码模式**：比赛可设置明文密码；未验证密码的用户进入比赛页/提交前须先提交密码（服务端校验，存储明文字段）。

## Capabilities

### New Capabilities

- `problem-access-guard`: 题目创建者、默认私有、比赛关联下的可见性矩阵与 API 鉴权规则。

### Modified Capabilities

- `problems`: 创建默认可见性、列表/详情鉴权、创建者字段。
- `contests`: 数字 ID、密码准入、管理端 CRUD 字段与时间语义、题目列表维护方式。
- `admin-ops`: 管理后台比赛弹窗与题目列表交互。

## Impact

- **数据库**：`Problem` 增加 `createdById`（或等价字段）、`Contest` 移除 `slug`、增加自增 `numberId`（或主键改为 Int）、`password`（可空）；迁移与现有比赛 slug 数据迁移策略。
- **API**：`problems.service` 列表/详情权限；`contests` 路由参数与 DTO；新增/调整密码验证端点；管理端 `admin/contests` 请求体。
- **Web**：`AdminContestPanel` 表单重构；`ContestPage` 密码门；路由由 slug 改为 id；题目/比赛链接全局替换。
- **文档**：开发者文档中比赛 URL 与可见性规则说明。
