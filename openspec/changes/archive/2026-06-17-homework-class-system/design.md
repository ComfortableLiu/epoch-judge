## Context

EpochJudge 面向高校和培训机构，但缺乏班级管理和作业系统。教师无法组织教学活动，学生无法跟踪作业进度。需要新增班级管理和作业系统。

## Goals / Non-Goals

**Goals:**
- 教师可创建班级，生成邀请码
- 学生通过邀请码加入班级
- 教师可布置作业（关联题目集 + 截止时间）
- 系统自动跟踪作业完成情况
- 教师可查看班级成绩统计

**Non-Goals:**
- 不做在线批改（自动评测已覆盖）
- 不做成绩导出（后续可扩展）
- 不做学生分组

## Decisions

### 1. 角色扩展
- **选择**：复用 PROBLEM_EDITOR 角色，添加 isTeacher 标志
- **理由**：最小化角色变更，Teacher 本质是特殊权限的用户
- **替代方案**：新增 TEACHER 角色——需要修改权限系统

### 2. 邀请码生成
- **选择**：6 位随机字母数字码
- **理由**：简单易记，足够唯一

### 3. 作业完成跟踪
- **选择**：基于 Submission 表实时计算
- **理由**：无需额外存储，数据实时准确

## Risks / Trade-offs

- [并发] 大量学生同时提交时性能压力 → 作业统计查询走 Redis 缓存
- [数据一致性] 学生退班后历史数据保留还是删除 → 保留，仅标记退班状态

## Migration Plan

1. 创建 Class、ClassMember、Homework、HomeworkProblem 表
2. API 新增 classes、homework 模块
3. 前端新增班级管理页、作业列表页
