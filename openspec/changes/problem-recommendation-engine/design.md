## Context

用户刷题时缺乏方向感，不知道下一步练什么。需要基于用户历史提交记录推荐合适的题目。

## Goals / Non-Goals

**Goals:**
- 基于提交历史计算用户能力画像
- 推荐难度略高于当前水平的题目
- 按标签差异化推荐薄弱领域
- 推荐结果缓存到 Redis

**Non-Goals:**
- 不做机器学习模型（纯规则引擎）
- 不做协同过滤推荐
- 不做个性化首页

## Decisions

### 1. 推荐算法
- **选择**：规则引擎——基于已通过最高难度 + 薄弱标签
- **理由**：简单可控，数据量不足时 ML 效果差

### 2. 缓存策略
- **选择**：Redis 缓存，TTL 1 小时
- **理由**：推荐结果不需要实时更新，1 小时足够

## Risks / Trade-offs

- [冷启动] 新用户无提交历史 → 推荐热门入门题目
- [准确性] 规则引擎可能推荐不准 → 后续可引入 ML 优化

## Migration Plan

1. API 新增 recommendations 模块
2. 前端题库页新增推荐区域
3. Redis 缓存推荐结果
